# VideoDecoder.cpp 注释和流程说明总结

## 文件概述
这是一个 FFmpeg 视频解码器的 C++ 实现文件，位于 libffplayer 模块中。

  - Android MediaCodec 硬件加速

### 3. 核心函数注释和流程说明

#### 3.1 `get_hw_format()` - 硬件格式回调
- 说明：获取硬件解码支持的像素格式回调函数
- 流程：遍历可用的像素格式，找到硬件支持的格式

#### 3.2 `prepare()` - 解码器准备
- **流程分为 9 个步骤**：
  1. ✅ 解析视频流参数（宽高、编码格式等）
  2. ✅ 选择合适的解码器（硬件优先）
  3. ✅ 初始化解码上下文
  4. ✅ 配置硬件加速（如果可用）
  5. ✅ 打开解码器
  6. ✅ 初始化帧缓冲和时间戳
  7. ✅ 收集媒体信息元数据

- **关键代码段注释**：
  - 硬件解码器选择逻辑
  - MediaCodec 配置过程
  - 降级到软件解码的处理

#### 3.3 `decode()` - 核心解码流程
- **流程分为 6 个主要步骤**：
  1. ✅ 判断是否为 EOF（End of File）标记
  2. ✅ 将编码包发送给解码器
  3. ✅ 循环接收解码后的帧
  4. ✅ 处理接收错误
  5. ✅ 处理有效的解码帧
  6. ✅ 处理像素格式转换

- **详细说明**：
  - AVERROR(EAGAIN) 处理
  - 重试逻辑说明
  - 像素格式处理逻辑
  - YUV420P10LE 到 RGBA 的转换

#### 3.4 `swsScale()` - 图像格式转换
- **流程分为 3 个步骤**：
  1. ✅ 初始化 SWS 上下文（首次使用时）
  2. ✅ 执行格式转换
  3. ✅ 更新目标帧信息

- **参数说明**：
  - 源宽度、高度、像素格式
  - 目标宽度、高度、像素格式
  - SWS_BICUBIC 插值算法说明

#### 3.5 `updateTimestamp()` - 时间戳更新
- **流程分为 4 个步骤**：
  1. ✅ 初始化同步起始时间（首次调用）
  2. ✅ 从帧中提取 DTS 或 PTS
  3. ✅ 时间戳转换（秒 -> 毫秒）
  4. ✅ Seek 后的时间戳修复

- **关键说明**：
  - av_q2d() 函数的用途
  - mTimeBase 时间基数含义
  - Seek 后的时间戳处理

#### 3.6 `avSync()` - 音视频同步
- **流程分为 4 个步骤**：
  1. ✅ 计算播放器实际经过时间
  2. ✅ 计算帧时间差值
  3. ✅ 限制最大延迟
  4. ✅ 等待或继续播放

#### 3.7 `seek()` - 快进/快退
- **流程分为 5 个步骤**：
  1. ✅ 清空解码缓冲
  2. ✅ 计算目标 Seek 位置
  3. ✅ 调用 FFmpeg Seek 函数
  4. ✅ 标记需要修复时间戳
  5. ✅ 记录 Seek 开始时间

- **标志说明**：
  - AVSEEK_FLAG_BACKWARD：优先查找关键帧
  - AVSEEK_FLAG_FRAME：逐帧定位

#### 3.8 `release()` - 资源释放
- **流程分为 6 个步骤**：
  1. ✅ 重置时间戳相关状态
  2. ✅ 释放视频帧缓冲
  3. ✅ 释放图像缩放上下文（libswscale）
  4. ✅ 释放 MediaCodec 上下文
  5. ✅ 释放硬件设备上下文
  6. ✅ 释放解码器上下文

- **注意**：此函数在析构函数中自动调用

#### 3.9 `getRotate()` - 获取旋转角度
- 说明：从视频流元数据中提取旋转信息
- 返回值：0, 90, 180, 270

#### 3.10 `getDisplayAspectRatio()` - 获取显示宽高比
- **流程分为 3 个步骤**：
  1. ✅ 获取样本宽高比（SAR）
  2. ✅ 计算显示宽高比（DAR）
  3. ✅ 化简比例

- **公式**：DAR = SAR × 视频宽 / 视频高

## FFmpeg 模块使用总结

### 涉及的 FFmpeg 库/模块
1. **libavcodec**：编解码库
   - `avcodec_find_decoder()` - 查找解码器
   - `avcodec_alloc_context3()` - 分配编码上下文
   - `avcodec_open2()` - 打开解码器
   - `avcodec_send_packet()` - 发送数据包
   - `avcodec_receive_frame()` - 接收解码帧

2. **libavformat**：格式/容器库
   - `avformat_seek_file()` - Seek 操作

3. **libswscale**：图像缩放和转换库
   - `sws_getContext()` - 获取转换上下文
   - `sws_scale()` - 执行转换

4. **libavutil**：工具库
   - `av_frame_alloc()` - 分配帧
   - `av_buffer_ref()` - 缓冲引用
   - 时间戳处理函数

### 硬件加速支持
- Android MediaCodec 硬件加速
- H.264 和 H.265(HEVC) 硬件解码

### 像素格式支持
- YUV420P（标准）
- YUV420P10LE（10-bit）
- NV12（硬件格式）
- RGBA（输出格式）
- MEDIACODEC（硬件格式）

## 使用建议

1. 此代码提供了完整的视频解码管道
2. 支持硬件加速能显著提高解码效率
3. 自动格式转换简化了渲染层的处理
4. 内置音视频同步机制确保播放质量
